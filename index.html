01 Origem

Blocos repetíveis (cardinalidade do XSD) têm botão para adicionar mais de um. * marca o que é obrigatório pelo XSD. Blocos opcionais começam recolhidos.

* obrigatório XSD XSD Archivematica recomendado

02 Modelo interno

Representação neutra das entidades. Fonte única de onde os dois perfis derivam. Vínculos por identificador local.

03 Saída + validação

O premis.xml gerado a partir do modelo, em dois perfis. A validação roda antes da exportação, com a fonte de cada regra.

        O * responde "preciso preencher?" (obrigatório no XSD). A cor responde "por ordem de quem?":

        (ex.: eventDateTime ISO 8601, agente por evento — o XSD não exige);
        recomendado nunca bloqueia. Um campo pode ser obrigatório só dentro de um bloco
        opcional: messageDigest só é exigido se você criar um fixity.
